Introduction

This file does all the prompting for the article “Gender Representation in Large Language Models: A Cross-Linguistic and Cross-Model Analysis”

Packages

We use Ollama with the R package rollama for the prompting. To make the prompting more efficient and make sure errors can be easily recovered from, I use a custom version of httr2::req_perform_parallel(), which rollama uses under the hood.

Main

We construct the full set of prompts here. To get only valid answers, we use the structured output technique, to limit the choices the model can make. We also run each combination of prompt and model 10 times, to add variation. In previous iterations, we noticed that Ollama does some caching of replies. So me make sure to first run all prompts in iteration 1, then all prompts in iteration 2 and so on.

all_prompts <- nouns |>
  left_join(prompts, by = c("lang" = "version")) |>
  mutate(
    prompt = paste(prompt, noun, sep = "\n"),
    # schema for structured outputs. Limits the possible answers the models can give
    schema = map(answers, function(a) {
      a <- str_split(a, ";")[[1]]
      list(
        type = "object",
        properties = list(
          association = list(
            type = "string",
            enum = a
          )
        ),
        required = "association"
      )
    })
  ) |>
  expand_grid(model = models, seed = 1:5, temperature = seq(0, 1, 0.2)) |>
  mutate(
    prompt_id = tolower(paste(lang, noun, model, temperature, seed, sep = "_")),
    .before = 1L
  ) |>
  mutate(prompt_id = str_remove(prompt_id, "/")) |>
  # for temperature 0, the seed does not change the outcome, so we don't need it
  filter(!(temperature == 0 & seed > 1L)) |>
  # arrange by model to make sure no unnecessary loading and unloading of models happen
  # but also arrange by seed to not run same prompt + model back to back
  arrange(seed, temperature, model)

I use rollama to prepare the requests to the servers, which are then made with the custom req_perform_parallel2 function defined above.

dir.create("cache", showWarnings = FALSE)
grid <- all_prompts |>
  mutate(
    req = pmap(
      list(prompt, model, schema, seed, temperature),
      function(p, m, s, sd, temp) {
        query(
          q = p,
          model = m,
          # we deactivated the structured output feature because thinking models (like gpt-oss) do not provide any answer at all
          # format = s,
          output = "httr2_request",
          model_params = list(seed = sd, temperature = temp)
        )[[1]]
      }
    )
  ) |>
  mutate(path = paste0("cache/", prompt_id, ".json"))

Note how much time was spent in total:

time_spent <- results |>
  pull(resp) |>
  map(function(r) {
    res <- jsonlite::read_json(r)
    tibble(
      model = pluck(res, "model"),
      total_duration = pluck(res, "total_duration"),
      eval_duration = pluck(res, "eval_duration")
    )
  }) |>
  bind_rows()

time_spent_summary <- time_spent |>
  group_by(model) |>
  summarise(
    total_duration = sum(total_duration, na.rm = TRUE),
    eval_duration = sum(eval_duration, na.rm = TRUE)
  ) |>
  arrange(desc(eval_duration)) |>
  mutate(
    total_duration = make_difftime(total_duration / 1e9, units = "h"),
    eval_duration = make_difftime(eval_duration / 1e9, units = "h")
  )
